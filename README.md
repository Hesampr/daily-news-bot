## 🧠 What It Does

Every morning, this bot automatically:

1. **Fetches** fresh articles from 6 curated sources across tech and startup media
2. **Filters** content using keyword and blacklist matching for relevance
3. **Deduplicates semantically** — if the same story appears across multiple sources, they are merged into one card using sentence-transformer embeddings
4. **Summarizes** each article into 4 sentences using Google Gemini AI
5. **Tags and categorizes** each article by field (AI & ML, Fintech, Health Tech, etc.)
6. **Delivers** everything directly to your Telegram — clean, structured, and ready to read

## ✨ Features

| Feature | Details |
|---|---|
| 📡 Multi-source fetching | Hacker News, TechCrunch, MIT Technology Review, Wired, The Verge, Product Hunt |
| 🧠 Semantic deduplication | Uses `sentence-transformers` (all-MiniLM-L6-v2) to detect and merge duplicate stories |
| 📝 AI summarization | 4-sentence summaries generated by Google Gemini 2.5 Flash |
| 🏷 Auto-tagging | 3–5 tags and field/category per article |
| 🔁 No duplicates across days | `seen_news.txt` tracks sent articles persistently |
| ⚙️ Fully automated | Runs daily via GitHub Actions — no server needed |
| 🛡 Robust error handling | Per-source failure isolation, automatic retries, graceful degradation |
| 💰 100% free | No paid APIs, no servers, no infrastructure cost |

## 🏗 Architecture

```
news-bot/
├── .github/workflows/
│   └── run.yml               # GitHub Actions scheduler (daily cron)
├── fetchers/
│   ├── hackernews.py         # Hacker News Algolia API
│   ├── rss_feeds.py          # TechCrunch, MIT, Wired, The Verge, Product Hunt (RSS)
├── processor/
│   ├── deduplicator.py       # Semantic similarity with sentence-transformers
│   └── summarizer.py         # Gemini 2.5 Flash summarization + tagging
├── bot/
│   └── telegram_sender.py    # Telegram message formatting and delivery
├── bot.py                    # Main orchestrator
├── config.py                 # Keywords, categories, blacklist, source config
└── seen_news.txt             # Persistent deduplication across runs
```

**Tech Stack:**

| Component | Technology |
|---|---|
| Language | Python 3.11 |
| Semantic similarity | `sentence-transformers` (all-MiniLM-L6-v2) |
| AI summarization | Google Gemini 2.5 Flash API (free tier) |
| RSS parsing | `feedparser` |
| Scheduling | GitHub Actions (cron) |
| Delivery | Telegram Bot API |

## 🚀 Setup Guide

### Step 1 — Fork this repository
Click **Fork** at the top right of this page to create your own copy.

---

### Step 2 — Create a Telegram Bot

1. Open [@BotFather](https://t.me/BotFather) on Telegram
2. Send `/newbot` and follow the prompts
3. Copy the **Bot Token** (format: `123456789:ABCdef...`)
4. Open [@userinfobot](https://t.me/userinfobot) to get your **Chat ID**
5. Send `/start` to your new bot to activate it

---

### Step 3 — Get a Gemini API Key

1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Sign in with your Google account
3. Click **Create API Key** and copy it

---

### Step 4 — Add GitHub Secrets

In your forked repo, go to **Settings → Secrets and variables → Actions → New repository secret** and add:

| Secret Name | Value |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Your bot token from BotFather |
| `TELEGRAM_CHAT_ID` | Your chat ID from userinfobot |
| `GEMINI_API_KEY` | Your Gemini API key from Google AI Studio |

---

### Step 5 — Enable Actions and Test

1. Go to the **Actions** tab in your forked repo
2. Click **I understand my workflows, enable them** if prompted
3. Select **News Bot 📰** from the left sidebar
4. Click **Run workflow → Run workflow**
5. Wait ~3 minutes and check your Telegram ✅

## ⚙️ Customization

### Topics & Filters
Edit `config.py` to control what news you receive:

```python
# Topics to include
INTEREST_KEYWORDS = ["startup", "AI", "innovation", "fintech", ...]

# Topics to exclude
BLACKLIST_KEYWORDS = ["celebrity", "sports", "gaming", ...]
```

### Schedule
Edit `.github/workflows/run.yml` to change the delivery time:

```yaml
- cron: "0 0 * * *"  # 8:00 AM Taiwan time (UTC+8)
```

Use [crontab.guru](https://crontab.guru) to generate your preferred cron expression.

### Number of Articles
Edit `MAX_ARTICLES_PER_RUN` in `config.py` (default: 15).

### Add or Remove Sources
Edit `RSS_SOURCES` in `config.py` to add any RSS feed:

```python
RSS_SOURCES = {
    "TechCrunch": "https://techcrunch.com/feed/",
    "Your New Source": "https://example.com/rss",
    ...
}
```

## 🛡 Error Handling

The bot is designed to never crash completely — partial results are always better than nothing.

| Scenario | Behavior |
|---|---|
| A source fails to fetch | Logged and skipped; other sources continue normally |
| Article missing title or link | Skipped silently |
| Article missing date | Delivered with "Unknown date" label |
| Semantic deduplication fails | Falls back to title-based deduplication automatically |
| Gemini API fails | Retries once after a delay; sends article without summary if retry fails |
| Telegram delivery fails | Retries once; skips and continues if retry fails |

## 🔖 Notes

- **GitHub Actions auto-disable**: GitHub disables scheduled workflows after 60 days of repository inactivity. Re-enable from the Actions tab if needed.
- **Gemini free tier**: Limited to a fixed number of requests per day. The bot includes rate limiting (7-second delay between calls) to stay within limits for up to 15 articles/day.
- **`seen_news.txt`**: Committed back to the repo after each run to track previously sent articles and prevent duplicates across days.
